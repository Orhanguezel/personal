Projedeki çoklu dil (i18n) altyapısını yönet: $ARGUMENTS

## Temel Prensipler

Bu projede çeviri sistemi **tamamen veritabanı tabanlıdır**. Deploy gerektirmeden:
- Yeni dil eklenebilmeli
- Mevcut dil kaldırılabilmeli
- Varsayılan dil değiştirilebilmeli
- Tüm çeviri metinleri admin panelden düzenlenebilmeli

## Veritabanı Şeması

Aşağıdaki tablo yapısını kullan (yoksa oluştur):

```sql
-- Desteklenen diller
CREATE TABLE languages (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  code        VARCHAR(10) NOT NULL UNIQUE,  -- 'tr', 'en', 'de'
  name        VARCHAR(100) NOT NULL,         -- 'Türkçe', 'English'
  native_name VARCHAR(100) NOT NULL,         -- 'Türkçe', 'English'
  direction   VARCHAR(3) DEFAULT 'ltr',      -- 'ltr' veya 'rtl'
  is_default  BOOLEAN DEFAULT false,
  is_active   BOOLEAN DEFAULT true,
  sort_order  INTEGER DEFAULT 0,
  created_at  TIMESTAMPTZ DEFAULT NOW(),
  updated_at  TIMESTAMPTZ DEFAULT NOW()
);

-- Çeviri anahtarları ve değerleri
CREATE TABLE translations (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  language_id   UUID NOT NULL REFERENCES languages(id) ON DELETE CASCADE,
  namespace     VARCHAR(100) NOT NULL,  -- 'common', 'auth', 'dashboard'
  key           VARCHAR(255) NOT NULL,  -- 'button.save', 'error.notFound'
  value         TEXT NOT NULL,           -- 'Kaydet', 'Save'
  is_verified   BOOLEAN DEFAULT false,
  created_at    TIMESTAMPTZ DEFAULT NOW(),
  updated_at    TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(language_id, namespace, key)
);

-- Hızlı erişim için index
CREATE INDEX idx_translations_lookup ON translations(language_id, namespace, key);
CREATE INDEX idx_translations_namespace ON translations(namespace);
```

## Görevler

Eğer yeni dil eklenmesi isteniyorsa:
1. `languages` tablosuna yeni kayıt ekle
2. Mevcut tüm çeviri anahtarlarını yeni dil için kopyala (değerleri boş bırak)
3. Admin panelde eksik çevirileri göster

Eğer yeni çeviri anahtarı ekleniyorsa:
1. Tüm aktif diller için kayıt oluştur
2. Varsayılan dil için değeri doldur
3. Diğer dilleri "çevrilmedi" olarak işaretle

Eğer UI'da hard-coded metin bulunursa:
1. Metni tespit et
2. Uygun namespace ve key belirle
3. Tüm diller için `translations` tablosuna ekle
4. Kodu `t('namespace.key')` formatına dönüştür

## Kontroller
- UI'da hiçbir hard-coded metin kalmamalı
- Tüm tarih/sayı formatları locale-aware olmalı
- Dil değiştiğinde sayfa yenilenmeden güncellenmeli
- Eksik çevirilerde fallback (varsayılan dil) çalışmalı
- SEO: Her dil için doğru `lang` ve `hreflang` attribute'ları olmalı
